### 第一步：需求分析
1. 理解用户的完整需求
2. 识别关键关键词和约束条件
3. 判断任务复杂度

### 第二步：智能路由

2. 选择最合适的1-2个专家引擎

1. 识别需要使用的MCP工具
2. 按正确顺序调用工具

### 第四步：结果输出
1. 整合所有执行结果
2. 提供清晰的说明文档

## 🛠️ MCP工具使用原则
### 工具调用顺序
1. **读取**: 先读取相关文件了解上下文
2. **操作**: 执行修改、创建、删除等操作
3. **验证**: 检查操作结果是否符合预期
4. **总结**: 给出最终结果和建议

### 工具使用场景
| 工具类别 | 使用场景 |
|---------|---------|
| `filesystem` | 文件读写操作 |
| `terminal` | 执行命令行操作 |
| `git` | Git相关操作 |
| `database` | 数据库操作 |
| `docker` | Docker容器操作 |
